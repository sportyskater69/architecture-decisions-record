[Title]
Hardware Components

[Status]
**What is the status, such as proposed, accepted, rejected, deprecated, superseded, etc.?**
Proposed


[Context]
**What is the issue that we're seeing that is motivating this decision or change?**
The Student Study Planner is designed to help students to plan their day, manage assignments and assign tasks. In order for this planner to work efficently, the hardware platofroms must suuport daily use, be capable of supporting real-time interaction and notifications. 

Student's need an application that can allow them to check deadlines, add notes, recieve reminders and have a access to the planner in any enviroment. 

[Decision]
**What is the change that we're proposing and/or doing?**
This application will run on android phones and iPhones. Smartphones were selected as primary devices since they are commonly used and carried throughout the day by their users. Compared to other devices such as computers and tablets, smartphones are more portable and easier to keep frequent interaction. Smartphones provide immediate access for users to check deadlines, take notes and get notifications which makes them a more practical choice compared to other hardware choices.

This application will use touchscreen, local storage and internet component as hardware. 

Touchscreen:
Touchscreen is the core hardware component needed for user interaction. This application relies on user input and adjustment. Using touchscreen users will be able to add information to the planner, mark tasks as completed, edit existing tasks and navigate between different sections. 

Local Storage:
The student study planner application will use the device’s local storage to store the user-entered data such as tasks, additional notes and deadlines. Local storage ensures that the data is saved even though user doesn’t have any access to the internet.

Internet:
Internet connection is optional for basic functionality since the application uses local storage. However, using internet may be useful for extended features. If internet is decided to get implemented it will be used for: Account creation and login process, cloud backup of user data, future updates and shared planning features.
